# specs/

This directory holds plans and decision records — the "why" behind a feature or chore before/while it's built (e.g. `specs/chore-{id}-{name}.md`).

Git history (and the archived pre-flatten repo) is the historical archive. Don't check generated screenshots, result dumps, or one-off audit output into `specs/` — those belong in a PR comment, a Linear ticket, or a scratch/log location outside version control.
