# 📖 อภิธานศัพท์ — คำที่เจอบ่อย (ภาษาคน)

> เปิดหน้านี้ค้างไว้ระหว่างเรียน เจอคำไหนไม่เข้าใจ กลับมาดูที่นี่ก่อน
> ทุกคำอธิบายสั้นๆ ไม่ต้องเข้าใจลึก แค่ "พอรู้ว่ามันคืออะไร" ก็อ่านบทต่อไปได้

---

## เครื่องมือ / การใช้งานทั่วไป

| คำ | แปลแบบบ้านๆ |
|----|-------------|
| **Terminal (เทอร์มินัล)** | ช่องจอดำๆ ที่เราพิมพ์คำสั่งคุยกับเครื่องโดยตรง บน Mac เปิดจาก Spotlight พิมพ์ "Terminal" |
| **command / คำสั่ง** | บรรทัดที่เราพิมพ์ในเทอร์มินัลให้เครื่องทำงาน เช่น `npm run dev` |
| **Node.js / node** | โปรแกรมที่ "รัน" โค้ด JavaScript/TypeScript บนเครื่องเรา |
| **npm** | ตัวจัดการ "ไลบรารี" (โค้ดของคนอื่นที่เรายืมมาใช้) + ตัวรันคำสั่ง `npm run ...` |
| **grep** | คำสั่ง "ค้นหาข้อความในไฟล์" เช่น `grep -rn "คำ" src/` = หาคำนี้อยู่ไฟล์ไหนบรรทัดไหน (`-r`=ทุกไฟล์, `-n`=บอกเลขบรรทัด, `-l`=บอกแค่ชื่อไฟล์) |
| **exit 0** | "รหัสจบงาน" ที่เครื่องคืนมาหลังรันคำสั่ง — `0` = สำเร็จ/ไม่มี error, เลขอื่น = มีปัญหา (มักไม่ต้องเห็นเลขตรงๆ ดูที่ข้อความ ✓ หรือ error แทน) |
| **build** | "สร้างเวอร์ชันจริง" ของเว็บ + ตรวจว่าโค้ดไม่มี error (`npm run build`) ด่านสำคัญก่อนส่งงาน |
| **compile** | แปลโค้ดที่เราเขียนให้เครื่อง/เบราว์เซอร์เข้าใจ (เกิดอัตโนมัติตอน build/รัน) |
| **Console / DevTools / F12** | "เครื่องมือนักพัฒนา" ในเบราว์เซอร์ กดปุ่ม `F12` (หรือคลิกขวา → Inspect) จะมีแผงเด้งขึ้น แท็บ **Console** คือที่ๆ error/warning ของหน้าโผล่ — เปิดดูเป็นนิสัยเวลาดีบัก |

## คำในโค้ด

| คำ | แปลแบบบ้านๆ |
|----|-------------|
| **import / export** | `export` = "เปิดให้ไฟล์อื่นเอาไปใช้", `import` = "ดึงของจากไฟล์อื่นมาใช้" เช่น `import { Button } from '@/components/humi'` = ขอ Button จากกล่อง humi มาใช้ |
| **barrel** | ไฟล์ `index.ts` ที่รวบรวม export ของหลายไฟล์ไว้ที่เดียว เพื่อให้ `import` จากที่เดียวสะดวก (เช่น `components/humi/index.ts`) |
| **component** | "ชิ้นส่วนหน้าจอ" 1 ชิ้น เขียนเป็นฟังก์ชันที่คืนหน้าตาออกมา (เช่น ปุ่ม การ์ด ทั้งหน้า) |
| **JSX** | การเขียน "หน้าตา HTML" ปนอยู่ในโค้ด เช่น `<h1>สวัสดี</h1>` ที่อยู่ใน .tsx |
| **prop (property)** | "ค่าที่ส่งเข้า component" เหมือนพารามิเตอร์ เช่น `<Button size="md">` — `size` คือ prop |
| **state** | "ข้อมูลที่ component จำได้" และเมื่อเปลี่ยน หน้าจอจะวาดใหม่ |
| **render / re-render** | "การวาดหน้าจอ" — re-render = วาดใหม่ (เกิดเมื่อ state/props เปลี่ยน) |
| **hook** | ฟังก์ชันพิเศษของ React ที่ขึ้นต้น `use` (เช่น `useState`, `useEffect`) ไว้เพิ่มความสามารถให้ component |
| **async / asynchronous** | "ทำงานแบบไม่รอ" — งานที่ใช้เวลา (เช่นเรียกข้อมูล) ทำไปเบื้องหลังแล้วค่อยแจ้งผลทีหลัง ไม่ค้างหน้าจอ |
| **JSON** | รูปแบบเก็บข้อมูลแบบ "กล่องซ้อนกล่อง" ใช้ปีกกา `{ }` เช่นไฟล์ `messages/th.json` (ระวังเครื่องหมายจุลภาค `,` — ดูบท 06) |
| **regex (regular expression)** | "รูปแบบค้นหา/จับคู่ตัวอักษร" หน้าตาแปลกๆ เช่น `/[0-9]/g` = "ตัวเลขทุกตัว" เจอในโค้ดปิดบังข้อมูล ไม่ต้องเขียนเองก็ได้ |

## Git / การส่งงาน

| คำ | แปลแบบบ้านๆ |
|----|-------------|
| **Git** | ระบบ "เซฟเกม" ของโค้ด บันทึก/ย้อนประวัติได้ |
| **commit** | "เซฟ" งาน ณ จุดหนึ่ง พร้อมข้อความอธิบาย |
| **branch (กิ่ง)** | สายงานแยก ทำของใหม่โดยไม่กระทบของหลัก (master) |
| **master** | กิ่งหลัก = ของที่ใช้จริง อย่าแก้ตรงๆ |
| **push** | ส่งงานขึ้น GitHub (เซิร์ฟเวอร์กลาง) |
| **PR (Pull Request)** | "ขอเอางานเรารวมเข้า master" ให้คนรีวิวก่อน merge |
| **merge** | รวมกิ่งของเราเข้า master (คนเป็นคนกด ไม่ใช่เราเอง) |
| **gh** | "GitHub CLI" เครื่องมือพิมพ์คำสั่งคุยกับ GitHub เช่น `gh pr create` (ต้องติดตั้ง + ล็อกอินครั้งแรก) |
| **Linear** | ระบบติดตามงาน (ticket) — งานหนึ่งใบมีคำอธิบาย + เงื่อนไขความสำเร็จ (AC) |
