# บท 11 — กฎเหล็กและกับดักที่ทำคนพลาดบ่อย

> เป้าหมาย: รวมกฎที่ห้ามฝ่าฝืน + กับดักที่ "เคยกินเวลาคนมาแล้ว"
> อ่านบทนี้ให้จบ แล้วกลับมาเปิดซ้ำเวลาติด

---

## ส่วน A: กฎเหล็ก (ห้ามฝ่าฝืน)

### 🔴 A1. ห้ามใช้สีแดง (NO-RED)
"อันตราย / ผิดพลาด / ลบ" ใช้ **ส้มฟักทอง (pumpkin) `--color-danger` `#FB923C`**
ห้าม: red, crimson, coral, clay, แดงเซ็นทรัล ทุกเฉด
> ทำไม: เป็นข้อกำหนดแบรนด์ของโปรเจกต์ ระบบมี "ตัวกัน" ที่จะเตือน/บล็อกถ้าเผลอใส่แดง

### 🎨 A2. ห้ามใส่สีดิบ / ขนาดดิบ
- ห้าม `#FF0000`, `color: red`, `text-[13px]` → ใช้โทเคน (`--color-...`) + คลาสมาตรฐาน (`text-sm`)
> เหตุผล: แก้ที่เดียวจบทั้งระบบ + ขนาดตัวอักษรจะ scale ตาม base ที่ตั้งให้ใหญ่เพื่อ HR

### 🌐 A3. สองภาษาครบเสมอ
เพิ่มข้อความ = เพิ่มทั้ง `th.json` + `en.json` (กุญแจเท่ากัน) ห้ามไทยปนใน en.json

### 🧱 A4. ใช้ของที่มีก่อนสร้างใหม่
หยิบชิ้นส่วน Cnext / store / helper / route ที่มีอยู่ ก่อนประดิษฐ์ใหม่ — diff เล็ก ย้อนง่าย

### 🔑 A5. เมนูไม่มีสิทธิ์ = เอาออก ไม่ใช่ล็อก (บท 07)

### 📥 A6. อนุมัติทุกอย่างรวมที่ `/quick-approve` (บท 07)

### 🔒 A7. ปิดบังข้อมูลอ่อนไหว
เลขบัญชี / เลขบัตรปชช. / เงินเดือน ต้องปิดบัง (mask) เป็นค่าเริ่มต้น
ใช้ helper: `maskValue` (`lib/date.ts`), `maskNationalId` (`lib/all-ported-employees.ts`), `maskTaxId` (`lib/tax-planning.ts`)
> ตัวอย่างจริง: `payroll-summary` ปิดเงินเดือนเป็น `฿••,•••` มีปุ่มตา (eye) ให้กดเผยทีหลัง

### 📅 A8. วันที่ใช้ พ.ศ.
อย่าจัดรูปแบบวันที่เอง ใช้ helper ใน `lib/date.ts` (`formatDate`) — มันแปลง พ.ศ. ให้ถูก
> เคยมีบั๊กจริง: คำนวณวันที่เองแล้วเครื่องหมายลบผิด ทำให้วันที่ในใบรับรองโชว์เป็น "-"

### 🚫 A9. เฟสนี้ไม่ต่อ backend จริง
อย่าเสียเวลาเขียน POST/PUT จริง ต่อ DB จริง — โฟกัสหน้าจอกดได้ + ข้อมูลจำลอง

### 🔧 A10. แก้แค่ `src/frontend/` (ไม่ยุ่ง `src/services/`)

## ส่วน B: กับดักที่เคยกินเวลา (อ่านก่อนตีหัวโต๊ะ)

### ⚠️ B1. แก้ CSS แล้วเบราว์เซอร์ไม่เปลี่ยน (แคชค้าง)
**อาการ:** แก้ `globals.css` แล้วสีไม่เปลี่ยน หรือหน้าเพี้ยน/ขาว
**สาเหตุ:** ตัว build (Turbopack) แคช CSS เก่าไว้
**แก้:** เซฟ `globals.css` ซ้ำอีก 1 ครั้ง (แก้อะไรนิดเดียวก็ได้) เพื่อบังคับ recompile
> ❗ อย่าสั่ง `rm -rf .next` (ถูกบล็อกในโปรเจกต์นี้) — ใช้วิธีเซฟซ้ำแทน
> เบราว์เซอร์ก็ลองกด `Cmd+Shift+R` (hard refresh) ด้วย

### ⚠️ B2. ลืม `'use client'`
**อาการ:** error เกี่ยวกับ `useState`/`onClick` ใช้ไม่ได้
**แก้:** เติม `'use client';` บรรทัดแรกสุดของไฟล์ (บท 02 §2.5)

### ⚠️ B3. ลืม `key` หรือ `key` ซ้ำ ใน `.map()`
**อาการ:** warning "same key" หรือลิสต์แสดงเพี้ยน
**แก้:** ใส่ `key` ที่ไม่ซ้ำ (id ก่อน, ถ้าไม่มีค่อย index) — ดูสูตร 5 บท 08

### ⚠️ B4. ลืม TITLE_MAP ตอนเพิ่มเมนู
**อาการ:** เพิ่ม leaf ในเมนูแล้วเทสต์ `cnext-functional` แดง
**แก้:** เพิ่มชื่อหน้าใน `TITLE_MAP` (`AppShell.tsx`) ทั้ง `/th/...` และ `/en/...`

### ⚠️ B5. ลืม locale ในลิงก์
**อาการ:** ผู้ใช้ภาษาอังกฤษกดลิงก์แล้วเด้งกลับไทย
**แก้:** ลิงก์ต้องเป็น `` `/${locale}/...` `` เสมอ (บท 06 §7)

### ⚠️ B6. JSON พัง (ลืมจุลภาค)
**อาการ:** ทั้งแอปขาว/พังหลังแก้ `messages/*.json`
**แก้:** เช็คจุลภาค `,` — ทุกบรรทัดในกล่องต้องมี ยกเว้นบรรทัดสุดท้าย (ห้ามมี)

### ⚠️ B7. รีเฟรชแล้วข้อมูลที่เพิ่มหาย
**ไม่ใช่บั๊ก** — mockup รีเซ็ตเป็น seed ตอนรีเฟรชโดยตั้งใจ (บท 05 §6)
ทดสอบ flow ข้ามหน้าโดยไม่รีเฟรช

### ⚠️ B8. พอร์ต 3000 ชน
ใช้พอร์ต 3000 เท่านั้น (8080 เป็นของ Keycloak) ถ้าชน = รันอยู่แล้ว เปิดเบราว์เซอร์ตรงๆ

## ส่วน C: ถ้าติดจริงๆ ทำยังไง

1. อ่านข้อความ error ให้ครบ — มันมักบอกชื่อไฟล์ + บรรทัด
2. เปิด Console (F12) ดู error สีแดง
3. `grep` คำที่เกี่ยวข้องหาจุดต้นตอ (บท 08 §6)
4. ดูกับดัก B1–B8 ข้างบน
5. `git diff` ดูว่าเราเพิ่งแก้อะไรไป (มักเป็นต้นเหตุ)
6. ย้อนกลับด้วย git ถ้าจำเป็น (บท 10 §4) แล้วลองใหม่ทีละขั้นเล็กลง
