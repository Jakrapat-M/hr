# บท 06 — ระบบสองภาษา ไทย/อังกฤษ (i18n)

> เป้าหมาย: เพิ่ม/แก้ข้อความบนหน้าจอได้อย่างถูกวิธี โดยรักษา "ความครบสองภาษา"
> นี่คืองานที่เคนจะทำบ่อยที่สุดงานหนึ่ง

---

## 1. หลักการ: ข้อความ "ไม่อยู่ในหน้า" แต่อยู่ในไฟล์ข้อความ

แทนที่จะเขียนข้อความตรงๆ ในหน้า เราเก็บข้อความไว้ที่เดียว 2 ไฟล์:
- `messages/th.json` — ไทย
- `messages/en.json` — อังกฤษ

แล้วในหน้าเราอ้าง "กุญแจ" (key) ของข้อความแทน ระบบจะหยิบภาษาที่ถูกมาแสดงตาม URL (`/th` หรือ `/en`)

> เครื่องมือที่ใช้ชื่อ **next-intl** — ตั้งค่าอยู่ที่ `src/i18n/` (ภาษา: ไทยเป็นค่าเริ่มต้น)

---

## 2. โครงสร้างไฟล์ข้อความ (กล่องซ้อนกล่อง)

`messages/th.json`:
```json
{
  "managerPayrollSummary": {
    "title": "สรุปค่าตอบแทนทีม",
    "eyebrow": "สวัสดีตอนเย็นค่ะ",
    "colName": "ชื่อ",
    "scope": "ผู้ใต้บังคับบัญชาโดยตรงของคุณ ({count} คน)"
  }
}
```

`messages/en.json` (ต้องมี "กุญแจเดียวกันเป๊ะ"):
```json
{
  "managerPayrollSummary": {
    "title": "Team compensation summary",
    "eyebrow": "Good evening",
    "colName": "Name",
    "scope": "Your direct reports ({count} people)"
  }
}
```

> `{count}` = ช่องเสียบค่าผันแปร (จะแทนด้วยตัวเลขจริงตอนแสดง)

---

## 3. เรียกใช้ข้อความในหน้า

3 ขั้น (ดูของจริงใน `manager/payroll-summary/page.tsx`):

```tsx
import { useTranslations } from 'next-intl';

export default function Page() {
  const t = useTranslations('managerPayrollSummary');  // 1) ชี้ไป "กล่อง" ที่จะใช้

  return (
    <div>
      <h1>{t('title')}</h1>                {/* 2) ดึงข้อความด้วยกุญแจ */}
      <p>{t('scope', { count: 10 })}</p>   {/* 3) เสียบค่า {count} */}
    </div>
  );
}
```
- `useTranslations('managerPayrollSummary')` = เปิดกล่องชื่อนี้
- `t('title')` = หยิบข้อความกุญแจ `title` (ได้ไทยหรืออังกฤษตาม URL อัตโนมัติ)
- `t('scope', { count: 10 })` = หยิบ + เสียบค่า

---

## 4. แบบที่ 2: เขียนสองภาษาในหน้าเลย (เจอบ่อยในระบบนี้)

บางหน้าไม่ใช้ไฟล์ข้อความ แต่เช็คภาษาในหน้าตรงๆ (ดู `time/corrections/page.tsx`):
```tsx
const locale = useLocale();
const isTh = locale !== 'en';   // true = ไทย

<h1>{isTh ? 'ขอแก้ไขเวลาเข้า-ออกงาน' : 'Request a time correction'}</h1>
```
อ่านว่า: ถ้าเป็นไทยแสดงข้อความไทย ไม่งั้นแสดงอังกฤษ

> ทั้ง 2 แบบใช้ได้ — แต่ **ต้องมีครบสองภาษาเสมอ** ไม่ว่าจะแบบไหน
> แบบไฟล์ข้อความ (`t(...)`) เหมาะกับข้อความเยอะ; แบบ `isTh ? :` เหมาะกับข้อความสั้นในหน้าเดียว

---

## 5. ⭐ กฎเหล็ก: "ครบคู่เสมอ" (TH/EN parity)

ทุกครั้งที่เพิ่มกุญแจใหม่ **ต้องเพิ่มทั้ง `th.json` และ `en.json`** ด้วยกุญแจชื่อเดียวกัน
- จำนวนกุญแจ 2 ไฟล์ต้อง "เท่ากันเป๊ะ" (ตอนนี้ ~4,637 กุญแจเท่ากัน)
- มีเทสต์ชื่อ `i18n-completeness` คอยจับ ถ้าไม่ครบคู่ → เทสต์แดง

> 🚫 ห้ามใส่ข้อความไทยในไฟล์ `en.json` (เทสต์จับได้ด้วย) — อังกฤษต้องเป็นอังกฤษจริง

---

## 6. ตัวอย่างเต็ม: เพิ่มข้อความใหม่ 1 ชิ้น (ทำตามได้)

สมมติอยากเพิ่มปุ่ม "ดาวน์โหลด PDF" บนหน้าหนึ่ง

**ขั้น 1** เปิด `messages/th.json` หากล่องที่เกี่ยวข้อง เพิ่ม:
```json
"downloadPdf": "ดาวน์โหลด PDF"
```
**ขั้น 2** เปิด `messages/en.json` กล่องเดียวกัน ตำแหน่งเดียวกัน เพิ่ม:
```json
"downloadPdf": "Download PDF"
```
**ขั้น 3** ในหน้า ใช้:
```tsx
<Button>{t('downloadPdf')}</Button>
```
**ขั้น 4** ดูบนเบราว์เซอร์ทั้ง `/th` และ `/en` ว่าถูกทั้งคู่

> ⚠️ JSON ระวังเครื่องหมายจุลภาค `,` — ทุกบรรทัดในกล่องต้องมี `,` ปิดท้าย
> ยกเว้นบรรทัดสุดท้ายของกล่อง (ห้ามมี `,`) ไม่งั้นไฟล์พัง

---

## 7. ลิงก์ข้ามหน้าต้อง "พกภาษาไปด้วย"

เวลาลิงก์ไปหน้าอื่น ต้องใส่ locale นำหน้าเสมอ (ดู `time/corrections/page.tsx`):
```tsx
<Link href={`/${locale}/time`}>เวลางาน</Link>
//          ↑ ใส่ /th หรือ /en ตามที่ผู้ใช้อยู่
```
ถ้าลืม → ผู้ใช้ที่อยู่ภาษาอังกฤษกดแล้วเด้งกลับไทย (พลาดบ่อย)

---

## สรุปบทนี้

- ข้อความอยู่ใน `messages/th.json` + `en.json` (หรือเขียน `isTh ? 'ไทย' : 'Eng'` ในหน้า)
- เรียกด้วย `const t = useTranslations('กล่อง')` แล้ว `t('กุญแจ')`
- **เพิ่มกุญแจต้องครบทั้ง 2 ไฟล์เสมอ** (มีเทสต์จับ)
- ลิงก์ข้ามหน้าต้องมี `/${locale}/` นำหน้า
