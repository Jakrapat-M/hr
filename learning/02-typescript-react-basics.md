> เราจะไม่สอนทุกอย่างในโลก — สอนแค่ที่ระบบนี้ใช้จริง โดยยกโค้ดจริงมาดู

## ส่วนที่ 1: TypeScript พื้นฐาน

TypeScript = JavaScript + "ระบบบอกชนิดข้อมูล" (type) ที่ช่วยกันพิมพ์ผิด

### 1.1 ตัวแปร (เก็บค่า)

```ts
const name = 'สมชาย';      // const = ค่าคงที่ เปลี่ยนไม่ได้ (ใช้บ่อยสุด)
let count = 0;             // let = เปลี่ยนค่าได้
count = count + 1;        // count เป็น 1 แล้ว
```

> กฎง่ายๆ: ใช้ `const` ไว้ก่อนเสมอ ใช้ `let` เฉพาะตอนที่ต้องเปลี่ยนค่าจริงๆ

### 1.2 ชนิดข้อมูล (type) — หัวใจของ TypeScript

```ts
const age: number = 30;            // ตัวเลข
const title: string = 'หัวหน้า';    // ข้อความ
const isActive: boolean = true;    // จริง/เท็จ
const tags: string[] = ['a', 'b']; // อาเรย์ (รายการ) ของข้อความ
```

ถ้าเราใส่ผิดชนิด TypeScript จะ "ขีดเส้นแดง" เตือนทันที เช่น
```ts
const age: number = 'สามสิบ';   // ❌ error! ใส่ข้อความในที่ที่ต้องเป็นตัวเลข
```
นี่แหละประโยชน์ — จับ bug ตั้งแต่ตอนพิมพ์ ไม่ต้องรอพัง

### 1.3 ออบเจ็กต์ (object) — ข้อมูลกลุ่มก้อน

```ts
const employee = {
  id: 'emp-001',
  name: 'สมชาย',
  age: 30,
};
console.log(employee.name);   // อ่านค่า → 'สมชาย'
```

ในระบบนี้เราใช้ **`type`** หรือ **`interface`** เพื่อกำหนด "หน้าตา" ของออบเจ็กต์ไว้ล่วงหน้า
ดูของจริงในระบบ — ไฟล์ `src/app/[locale]/manager/payroll-summary/page.tsx`:

```ts
interface CompRow {
  readonly emp: HumiEmployee;   // พนักงาน
  readonly base: number;        // เงินเดือนฐาน
  readonly allowances: number;  // เบี้ยเลี้ยง
  readonly total: number;       // รวม
}
```
แปลว่า "CompRow ทุกตัวต้องมี 4 ฟิลด์นี้ ครบและชนิดถูกต้อง"
(`readonly` = ตั้งค่าแล้วแก้ไม่ได้ กันพลาด)

### 1.4 ฟังก์ชัน (function) — กล่องรับเข้า-ส่งออก

```ts
function add(a: number, b: number): number {
  return a + b;
}
add(2, 3);   // ได้ 5
```
อ่านว่า: ฟังก์ชัน `add` รับ `a` (ตัวเลข) กับ `b` (ตัวเลข) แล้วคืน (return) ตัวเลข

แบบสั้น (arrow function) — เจอบ่อยมากในระบบ:
```ts
const add = (a: number, b: number) => a + b;
```

ตัวอย่างจริงจาก `payroll-summary/page.tsx` — ฟังก์ชันปิดบังตัวเลขเป็นจุด:
```ts
function maskCurrency(formatted: string): string {
  return formatted.replace(/[0-9๐-๙]/g, '•');   // แทนทุกตัวเลขด้วย •
}
// '฿55,000' → '฿••,•••'
```

## ส่วนที่ 2: React พื้นฐาน

React = วิธีสร้างหน้าจอจาก "ชิ้นส่วน" เล็กๆ เรียกว่า **component**

### 2.1 Component คืออะไร

Component = ฟังก์ชันที่ "คืนหน้าตา" (UI) ออกมา
หน้าตาเขียนด้วย **JSX** (หน้าตาเหมือน HTML แต่อยู่ในโค้ด)

```tsx
function Hello() {
  return <h1>สวัสดีครับ</h1>;
}
```
อ่านว่า: component ชื่อ `Hello` คืนหัวข้อ `สวัสดีครับ`
เอาไปใช้: `<Hello />` (เหมือนสร้างแท็กใหม่ของเราเอง)

> 📌 ชื่อ component **ขึ้นต้นด้วยตัวใหญ่เสมอ** (`Hello` ไม่ใช่ `hello`)
> 1 ไฟล์ `page.tsx` = 1 หน้าเว็บ ข้างในคือ component ใหญ่ 1 ตัว

### 2.2 JSX — เขียน HTML ในโค้ด

```tsx
return (
  <div className="space-y-6">
    <h1>หัวข้อ</h1>
    <p>เนื้อหา</p>
  </div>
);
```

จุดต่างจาก HTML ที่ต้องรู้:
- ใช้ **`className`** แทน `class` (เพราะ `class` เป็นคำสงวนใน JS)
- ใส่ค่าจาก JS ด้วย **ปีกกา `{ }`** เช่น `<h1>{title}</h1>`
- แท็กที่ไม่มีเนื้อหาต้องปิดตัวเอง เช่น `<img />`, `<Clock3 />`

ตัวอย่างจริง — เอาค่าจากตัวแปรมาแสดง:
```tsx
<p className="text-ink-muted">
  {t('scope', { count })}      {/* เอาข้อความ + จำนวนมาแสดง */}
</p>
```

### 2.3 Props — ส่งค่าเข้า component

Props = "พารามิเตอร์" ของ component (ส่งข้อมูลเข้าไป)

```tsx
function Greeting({ name }: { name: string }) {
  return <h1>สวัสดี {name}</h1>;
}

// เอาไปใช้ — ส่ง name เข้าไป
<Greeting name="สมชาย" />     // แสดง: สวัสดี สมชาย
```

ในระบบ ชิ้นส่วน Humi รับ props แบบนี้ตลอด:
```tsx
<Button variant="primary" size="md" onClick={handleSubmit}>
  ส่งคำขอ
</Button>
```
อ่านว่า: ปุ่มแบบ primary ขนาด md พอกดให้เรียก `handleSubmit` ข้อความปุ่มคือ "ส่งคำขอ"

### 2.4 State — ข้อมูลที่ "จำได้" และทำให้จอเปลี่ยน

`useState` = ตัวเก็บค่าที่เมื่อเปลี่ยน หน้าจอจะวาดใหม่เอง
ตัวอย่างจริงจาก `payroll-summary/page.tsx`:

```tsx
const [revealed, setRevealed] = useState(false);
//     ↑ค่าปัจจุบัน  ↑ฟังก์ชันเปลี่ยนค่า   ↑ค่าเริ่มต้น = false (ซ่อนอยู่)

// พอกดปุ่ม สลับ true/false → จอเปลี่ยนระหว่าง "ซ่อน" กับ "โชว์ตัวเลข"
<Button onClick={() => setRevealed((v) => !v)}>
  {revealed ? 'ซ่อน' : 'แสดง'}
</Button>
```
อ่านว่า:
- `revealed` คือค่าปัจจุบัน (จริง/เท็จ)
- `setRevealed(...)` คือวิธีเปลี่ยนค่า
- `revealed ? 'ซ่อน' : 'แสดง'` = ถ้าจริงแสดง "ซ่อน" ถ้าเท็จแสดง "แสดง" (เงื่อนไขแบบสั้น)

> 💡 กฎเหล็ก: **เปลี่ยน state ต้องเรียกฟังก์ชัน `setXxx` เท่านั้น** ห้ามแก้ตัวแปรตรงๆ
> ไม่งั้น React ไม่รู้ว่าต้องวาดจอใหม่

### 2.5 `'use client'` — บรรทัดลึกลับบนหัวไฟล์

หลายไฟล์ขึ้นต้นด้วย:
```tsx
'use client';
```
แปลว่า "หน้านี้ทำงานฝั่งเบราว์เซอร์ (มีการกด/พิมพ์/จำ state)"

กฎง่ายๆ:
- หน้าที่ **มีปุ่มกด / ฟอร์ม / useState / onClick** → ต้องมี `'use client'` บนสุด
- หน้าที่ **แค่แสดงผลเฉยๆ ไม่มีการโต้ตอบ** → ไม่ต้องมี (เร็วกว่า)

ถ้าลืมใส่แล้วใช้ `useState` จะ error — แค่เติม `'use client';` บรรทัดแรกของไฟล์

### 2.6 `.map()` — วาดรายการซ้ำๆ (สำคัญมาก)

เวลามีลิสต์ (เช่นรายชื่อพนักงาน 10 คน) เราใช้ `.map()` วาดทีละแถว:
```tsx
{employees.map((emp) => (
  <div key={emp.id}>{emp.name}</div>
))}
```
อ่านว่า: "สำหรับพนักงานแต่ละคน วาด `<div>` ที่แสดงชื่อ"

> ⚠️ **`key` สำคัญมาก** — แต่ละแถวต้องมี `key` ที่ **ไม่ซ้ำกัน** (นิยมใช้ `id`)
> React ใช้ key จำว่าแถวไหนเป็นแถวไหน ถ้า key ซ้ำจะเตือน/เพี้ยน
> (bug จริงที่เพิ่งแก้ไปในระบบนี้คือ key ซ้ำ — ดูตัวอย่างเต็มใน [บท 08](./08-maintenance-recipes.md))
