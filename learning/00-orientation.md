# บท 00 — ระบบนี้คืออะไร (ปฐมนิเทศ)

> บทนี้ไม่มีโค้ดเลย เป้าหมายคือให้ "เห็นภาพรวม" ก่อน เหมือนดูแผนที่ก่อนออกเดินทาง

---

## 1. ระบบนี้คืออะไร

เป็น **เว็บแอป HR** (ระบบบุคคล) สำหรับพนักงานและฝ่าย HR ใช้งาน เช่น
- พนักงานดูโปรไฟล์ตัวเอง / ขอลา / ดูสลิปเงินเดือน / แก้เวลาเข้า-ออกงาน
- หัวหน้าอนุมัติคำขอของลูกทีม / ดูสรุปค่าตอบแทนทีม
- HR ดูทะเบียนพนักงาน / จัดการสวัสดิการ / ออกรายงาน

ตอนนี้ระบบอยู่ในเฟส **"ตัวอย่างหน้าจอที่กดได้จริง" (clickable UI mockup)** —
หน้าจอเปิดได้ ตารางแสดงข้อมูล ฟิลเตอร์ทำงาน กดไปมาระหว่างหน้าได้
แต่ **ยังไม่ได้เชื่อมฐานข้อมูล/ระบบหลังบ้านจริง** ข้อมูลที่เห็นเป็น "ข้อมูลจำลอง" ที่เขียนฝังไว้ในโค้ด

> 💡 ทำไมต้องทำแบบนี้? เพื่อให้ทีม HR ได้ลองกดเล่นจริง เห็นหน้าตา/การไหลของงาน
> แล้ว "เคาะ" ทิศทางก่อน ค่อยลงทุนทำของหลังบ้านจริงทีหลัง

---

## 2. ภาพรวมเทคโนโลยี (รู้จักชื่อไว้พอ ยังไม่ต้องเข้าใจลึก)

ลองนึกภาพเว็บเป็น "บ้าน" แต่ละอย่างทำหน้าที่ต่างกัน:

| ชื่อ | คืออะไร | เปรียบเทียบ |
|------|---------|-------------|
| **Next.js** | โครงเว็บหลัก จัดการว่า URL ไหนแสดงหน้าไหน | โครงสร้างบ้าน + ระบบเดินทางในบ้าน |
| **React** | ตัวสร้าง "หน้าจอ" จากชิ้นส่วนเล็กๆ (component) | อิฐ/บล็อกที่เอามาประกอบเป็นห้อง |
| **TypeScript** | ภาษาเขียนโค้ด (JavaScript + ระบบกันพิมพ์ผิด) | ภาษาพูดในบ้าน + ยามที่คอยเตือนเวลาทำผิด |
| **Tailwind CSS** | ระบบแต่งสี/ขนาด/ระยะห่าง ด้วยคำสั้นๆ | สีทาผนัง + เฟอร์นิเจอร์ |
| **Cnext** | ชุดดีไซน์ของเราเอง (ปุ่ม การ์ด ตาราง ที่ทำไว้ให้แล้ว) | ชุดเฟอร์นิเจอร์สำเร็จรูปประจำบ้าน |
| **Zustand** | ที่เก็บข้อมูลชั่วคราว "จำได้ระหว่างที่ยังเปิดเว็บ" | กระดานโน้ตกลางบ้าน |
| **next-intl** | ระบบสองภาษา ไทย/อังกฤษ | ป้ายสองภาษาทุกห้อง |

ไม่ต้องจำตอนนี้ — แต่ละตัวมีบทของมันเอง (บท 02, 04, 05, 06)

---

## 3. "หน้าหนึ่งหน้า" ประกอบจากอะไร

เวลาเปิดหน้าเว็บหนึ่งหน้า เช่น "ขอแก้ไขเวลา" มันมาจากของพวกนี้รวมกัน:

```
┌─────────────────────────────────────────────────────┐
│  URL: /th/time/corrections                          │
│  ↓ Next.js แปล URL นี้ไปหาไฟล์...                     │
│  src/app/[locale]/time/corrections/page.tsx          │  ← "หน้า" (page)
│      ใช้ ↓                                            │
│  ชิ้นส่วน Cnext: <Card> <Button> <FormField>          │  ← "เฟอร์นิเจอร์สำเร็จรูป"
│      ข้อความจาก ↓                                     │
│  messages/th.json + messages/en.json                 │  ← "ป้ายสองภาษา"
│      ข้อมูล/สถานะจาก ↓                                │
│  src/stores/time-corrections.ts (Zustand)            │  ← "กระดานโน้ต"
│      สี/ระยะจาก ↓                                     │
│  src/app/globals.css (design tokens) + Tailwind      │  ← "สีและการตกแต่ง"
└─────────────────────────────────────────────────────┘
```

ทุกหน้าในระบบมีโครงแบบนี้หมด พอเข้าใจหน้าเดียว = เข้าใจทั้งระบบ

---

## 4. กฎสำคัญ 3 ข้อที่ต้องรู้ตั้งแต่วันแรก

1. **แก้แค่ใน `src/frontend/`** — โฟลเดอร์อื่นเป็นของเก่า/อ้างอิง อย่าไปยุ่ง
2. **ภาษาเริ่มต้นคือ "ไทย"** — URL จะมี `/th/` นำหน้า (อังกฤษคือ `/en/`) ทุกข้อความต้องมีครบ 2 ภาษาเสมอ
3. **ห้ามใช้สีแดง** — ระบบนี้ใช้สีส้มฟักทอง (pumpkin) แทนแดงสำหรับ "อันตราย/ผิดพลาด" (เหตุผลใน [บท 11](./11-golden-rules-and-traps.md))

---

## 5. สรุปบทนี้

- ระบบนี้ = เว็บ HR แบบ "ตัวอย่างกดได้" ยังไม่ต่อหลังบ้านจริง ข้อมูลเป็น mock
- ของที่เราดูแลอยู่ใน `src/frontend/`
- 1 หน้า = page.tsx + ชิ้นส่วน Cnext + ข้อความ 2 ภาษา + กระดานโน้ต Zustand + สี/โทเคน
- จำกฎ 3 ข้อ: แก้แค่ frontend / สองภาษาเสมอ / ห้ามแดง
