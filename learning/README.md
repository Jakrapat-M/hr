## เคล็ดลับการเรียน

1. **อย่ารีบ** — บท 00–03 ปูพื้นไว้ ถ้าข้ามไปแก้โค้ดเลยจะงง
2. **เปิดโค้ดจริงคู่กันไป** — ทุกครั้งที่บทอ้างถึงไฟล์ ให้เปิดไฟล์นั้นดูด้วยตา
3. **แก้ของเล็กๆ ก่อน** — เปลี่ยนข้อความ เปลี่ยนสี ดูผลบนเบราว์เซอร์ จะสนุกและเห็นภาพ
4. **พังได้ ไม่เป็นไร** — git ช่วยย้อนกลับได้เสมอ (บท 10) เครื่องนี้พังยาก
5. **ถ้าติด** — ดูบท 11 "กับดัก" ก่อน ปัญหาที่เจอบ่อยรวมไว้แล้ว

## สิ่งที่ต้องรู้ก่อนเริ่ม (อ่าน 1 นาที)

- **ระบบนี้คือ "ตัวอย่างหน้าจอ" (UI mockup)** ที่กดได้จริง ใช้ให้ทีม HR ดูทิศทางก่อนทำของจริง

- **โค้ดที่เราดูแลอยู่ที่เดียว: `src/frontend/`** — ส่วนอื่น (`src/services/*`) เป็นของเก่า ไม่ต้องยุ่ง
